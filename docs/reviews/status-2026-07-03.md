## 3. UI review (senior-UX pass, as served today)

**What's genuinely good**
- One coherent product: shared tokens, wordmark, icon set, and accessible drawer nav across all five v6 surfaces; old UIs are redirect stubs, so there is no longer a second front door serving stale numbers.
- The hero page renders the full answer stack: augment lead → Eternal major + both minor pickers → tabbed meta builds + sim build with per-item "why" → matchups with counter-swaps → Learn tab. Sim-vs-field agreement is honestly badged (⚙/✓) instead of hidden.
- Coach film UI is the standout: linked timeline↔map↔fight breakdown, kill heatmap on real map geometry, fight-economics panel, per-member reports. This is differentiated work no other Predecessor tool has.
- Honesty is a UI feature: THEORY labels, the patch-page "predictions, not measurements" banner, and About's self-declared gaps all match the engine's actual state.

**Gaps, ranked**
1. **`ui/learn-eternals.html` is the one off-brand served page** — no `nav.js`, no `icons.js`, old styling, still linked from the drawer. Either restyle into v6 or fold into the hero page's Learn tab.
2. **Field-evidence labels are now wrong-ish**: hero page says "by field winrate" in 4 places, but since 07-02 the evidence is ranked-only, version-pinned. Relabel is trivial and honesty-relevant (ties to priorities #12).
3. **`patch-1.15.html` lacks the shared drawer** — it has its own chapter subnav but no way back into the rest of the app except the wordmark.
4. **Emoji remnants** undercut the brand pass: load-bearing semantics (⚙ sim, 🧠 AI-written, 👑/💤/⚠ meta badges) and Live Draft's lane emojis are still emoji, exactly the "mush at 15px" problem the icon set was built to fix.
5. **Computed-but-never-shown evidence**: `agreement-audit.json` (coreRecall 0.583) and the leave-one-out `explain` output have no UI consumer (v6-review D2 + priorities #11). The "why the field's build wins" panel is the single most valuable missing UI feature — it converts the sim from a parallel opinion into an explainer of reality.

## 6. Method note

The month's most important process lesson (lessons.md, 07-02/07-03): grounding
the *numbers* doesn't ground the *narrative* — the author invents connective
tissue only the independent critic catches, and every source enrichment makes
old converged copy newly checkable. The author→judge→gate loop is not
optional for fresh authoring, and the corpus should be re-swept after any
fact-schema enrichment. That discipline (plus "introspect what we already
hold before assuming a re-pull") is now the repo's core methodology and is
documented in `docs/agent-loops.md`.
